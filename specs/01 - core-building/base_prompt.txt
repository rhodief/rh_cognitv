This a brief draft for my new cognitive-driven agentic framework. The general idea is to have 3 layers which can be extended and replaced during the development process, cause they are related by kinda indepent. As you can see, layer 3 is the execution platform, they statnd alone and should be built by dependency invertion. The layer two rely on layer 3, but there're especifics adapter to transform their data into layer 3 data. The same for layer 01. The cognitive layer do not touch layer 3, they do it via layer 2 and vice-versa. In the first version, the idea is write the skills elements and apply it to orchestration, which will have many way to design throw planning and execution DAG's, but in the future, the idea is create the whole orchestration declaratively, which a "metaskill" will help users to create new skills and orchestrations. One of the main "why's" behing this project is to have full control of the orchestration in order to have harness and reliable agents, with reach logs and trace of each execution path, allowing the user UNDO and REDO stuff with a powerfull time-travel capabilities.
